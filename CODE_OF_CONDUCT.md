# Code of Conduct

## Our standard

Jamal's Finance welcomes focused, respectful collaboration. Participants are expected to communicate clearly, critique ideas rather than people, respect project scope, and protect the privacy of anyone whose data could appear in a report, screenshot, fixture, or discussion.

Examples of positive participation include:

- Giving actionable, evidence-based feedback
- Respecting documented security, data-integrity, and accessibility boundaries
- Keeping public discussions free of credentials, personal data, and private financial records
- Accepting maintainer decisions about roadmap order, scope, and release readiness
- Correcting mistakes transparently

Unacceptable behavior includes harassment, discriminatory language, personal attacks, deliberate disruption, publishing another person's private information, or using the project to test unauthorized access to accounts or data.

## Scope

This code applies in repository issues, pull requests, reviews, discussions, and any other project space represented by the maintainer.

## Enforcement

Report conduct concerns privately to the repository owner through GitHub. Do not open a public issue when doing so would expose personal, security-sensitive, or identifying information.

The maintainer may edit or remove contributions and may temporarily or permanently restrict participation when behavior is unsafe, abusive, misleading, or materially disruptive to the project.
